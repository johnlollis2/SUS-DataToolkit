The System Usability Scale (SUS) provides a reliable tool for measuring usability. It consists of a 10-item questionnaire with five response options for respondents; from Strongly agree to Strongly disagree. The SUS score is then calculated by converting the raw scores to a 0-100 scale.

The median is a measure of central tendency that represents the middle value in a data set when it is ordered from least to greatest. In the context of SUS scores, the median helps identify the central score in a sample, providing insight into the typical user experience.

Please read more

Standard deviation measures the amount of variation or dispersion in a set of values. A low standard deviation indicates that the values tend to be close to the mean, whereas a high standard deviation indicates that the values are spread out over a wider range. For SUS scores, it helps assess the consistency of user responses.

Please read more

Adjective ratings provide a qualitative measure that describes the usability of a system in terms of user satisfaction. Common adjectives used include "Excellent," "Good," "Fair," "Poor," and "Awful." These ratings help to contextualize SUS scores in a way that is easily understandable.

Please read more

SUS scores can be converted into letter grades (A to F) to provide a more intuitive understanding of the results. This conversion is based on the percentile ranks of SUS scores. For example, scores above 85 are considered "A" grade, indicating excellent usability.

Please read more

Acceptability refers to the degree to which users find a system to be usable and satisfactory. SUS scores are often categorized into levels of acceptability, such as "Not Acceptable," "Marginally Acceptable," and "Acceptable," helping to quickly assess whether a system meets user expectations.
